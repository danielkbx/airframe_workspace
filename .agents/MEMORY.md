- Main-stream resync doctrine (2026-07-13): a rejected `I` frame becomes a tentative absolute anchor. Airframe decodes and buffers it plus following main frames, publishing the buffer only after two consecutive `P` frames satisfy normal monotonic jump checks. An out-of-window `I` may provide untrusted predictor state while the parser searches for a later local anchor, but neither it nor its `P` frames are exposed or allowed to extend the accepted timeline. A tentative anchor must remain within six normal time-jump windows of the last committed frame; typed `loggingResume` remains the only legitimate discontinuity. On failure, committed history is restored; `P` frames never anchor by themselves and unknown events never seed recovery. On the private 8.0 MiB Betaflight 4.5.2 Maya log, this recovers at byte `1,173,982` / `23.035302 s` and ends log 1 at `60.541099 s` (Upstream: `00:08–01:00`), rejecting the false ~`693 s` continuation. Log 2 is independently `104.486025–127.427661 s` (Upstream: `01:44–02:07`). Defects are user-visible: `ReaderLogIndex.nearestMainFrameTime(atOffset:)` maps issue offsets to times, timeline renders issue markers (orange `Style.issueLine`) next to event markers with shared cap and same-time dedupe.
- One-pass scan doctrine (user directive 2026-07-13, implemented in commits `3b61d63`, `046ac33`, `768ef17`, `dec5496`, `430969b`): the open scan is THE data-acquisition step and must collect everything whole-flight consumers need in one decode pass. `DecodedLogScan.overview: ReaderScanOverview` carries decimated main-frame samples (FULL field layout per sample, adaptive stride halving, budget `Configuration.maxScanOverviewSampleCount` = 1024, final count in [budget, 2x budget)) plus all `ReaderEventRecord`s (truncating at `maxEventRecordCount` with a flag instead of throwing). New series (gyro, RPM, ...) are added as projections in `BlackboxAnalysisWorkspace.overviewPoints(for:in:)` — never as another decode pass. Full-resolution zoomed views remain range queries. `summarize()`/`makeIndex()` skip overview collection and are unchanged. Concurrency doctrine: one frame stream is sequential; independent logs of a document scan in parallel (task group capped by `maxConcurrentLogTasks`); projections over the immutable Sendable overview are parallelizable by construction. Release numbers (5.7 MB log): scan 0.77 s -> 0.54 s (per-frame dictionary removal) -> 0.57 s with overview (~5% overhead); timeline data 1.2 s (viewport+events) -> 5.6 ms (projection).
- Concurrency rule (user directive 2026-07-12): data preparation must always run off the main actor, and high-frequency progress callbacks must be throttled before they reach main-actor state. Heavy worker functions must never live as members of SwiftUI View types: `View` is a MainActor protocol, its isolation propagates to all members, and detached tasks calling them synchronously execute on the main actor (verified by process sample 2026-07-13). Declare such builders `nonisolated` or keep them on non-View types. `DocumentProgressRelay` forwards roughly 256 byte-delta-gated updates per log; per-frame forwarding stalled the main thread (beachball) for entire scans of multi-megabyte logs. Never access `DecodedLog.headerInfo`/`seriesCatalog` (computed, header reparse per access) inside per-row loops; hoist into a per-request context (see `BlackboxAnalysisWorkspace.RowContext`).
- UI-facing numeric text should use proper number formatters instead of raw string interpolation, `String(format:)`, or `String.init` for numbers. - Per-window UI state decision (2026-07-12): view selection, macOS inspector visibility, and sidebar column visibility live in `DocumentStateStore` (`App/Airframe/App/DocumentStateStore.swift`, `@Observable`, injected via `EnvironmentValues.documentStateStore`). - Per-document UI state decision (2026-07-12): documents reopen in their last state via a rolling buffer in UserDefaults (`DocumentStateRepository`, key `airframe.documentStates`, cap 50, LRU eviction by `updatedAt`), keyed by `LogDocument.contentIdentity` = `v1:` + SHA-256 over byte count + first/last 64 KiB (middle intentionally not hashed; version prefix allows strategy changes). Read order at open: document entry supersedes globals (`airframe.logView.selection`, `airframe.logView.inspectorPresented`, `airframe.document.columnVisibility`) supersedes hard defaults. `persist()` writes NOTHING when the raw-value snapshot equals the loaded baseline; on change it writes globals AND the document entry, then resets the baseline (idempotent double-calls). xattr-on-file was considered and REJECTED: blackbox logs live on FAT32/exFAT SD cards where xattrs do not survive, the app is a viewer and must not touch documents, and a future sandbox would block the writes; the content hash even survives FAT32 copies. iCloud sync of the buffer is planned next; `DocumentStateRepository` is the single seam for the NSUbiquitousKeyValueStore mirror (app currently has no entitlements). Exact pixel-level persistence of manually dragged sidebar widths remains a possible later macOS-specific enhancement only if SwiftUI's built-in split restoration is insufficient.
